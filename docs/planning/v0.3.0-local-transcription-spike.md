# v0.3.0 — Local Transcription Spike

## Goal

Prove the first real local transcription path for shared WhatsApp `.opus` voice notes before integrating a native `whisper.cpp` stack.

## Reference sample

Validated `v0.2.4`/`v0.2.5` share protocol:

- file: `PTT-20260608-WA0000.opus`
- format: `OGG_OPUS`
- MIME: `audio/ogg; codecs=opus`
- extension: `opus`
- size: `652 KB`
- duration: `4:45 min`
- header probe: `32768 Bytes`
- guardrails: warning active, within soft/hard limits

## v0.3.0 implementation

1. Accept the existing shared audio URI.
2. Validate ingest policy and duration guardrails.
3. Decode audio via Android media stack.
4. Convert decoded PCM to 16 kHz mono PCM.
5. Pass the PCM file descriptor to Android's on-device file recognizer when available.
6. Show transcript if successful.
7. Fall back to concise ingest/guardrail output if unsupported or failed.

## Known constraints

- This is a spike, not the final STT engine.
- Android on-device file recognition support depends on the device and recognizer service.
- `whisper.cpp` native integration remains the next hardening step.
- No cloud STT is intentionally added.
